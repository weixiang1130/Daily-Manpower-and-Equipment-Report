# Changelog

版本異動摘要。完整背景與設計決策請見 [`docs/milestones/`](docs/milestones/README.md)。

## [節點 06] 2026-07-02 — Git / GitHub 版本控管建立
- 建立 git 版控與 GitHub 遠端，`main` 為預設分支，改版一律走 PR 流程
- 程式碼庫預設清單去識別化為通用範例值
- 新增 `config.local.js` 機制：地端真實名單自動載入、永不入版控

## [節點 05] 2026-07-02 — 多工地列控支援
- 工地欄位改為申請表單內選定（移除頁首全域工地選單）
- 匯入 16 個實際列控工地
- 總覽新增跨工地彙總卡片與「各工地列控總覽」表格，可點擊跳轉篩選
- 點工清單、機具清單、歷程報表新增工地篩選器

## [節點 04] 2026-07-02 — Excel 欄位對齊與大頁籤重構
- 點工申請/回報、機具申請/回報改為「大頁籤 + 申請／回報子模組」，兩者為真正獨立的表單
- 回報欄位改為完全對齊既有 Excel 查核表（點工出工列控 / 機具出工列控）
- 移除頁首「目前填報工程師」，改由申請人 / 簽單責任工程師各自於對應表單選擇
- 工作地點、工作內容類別、機具類型改為「單行下拉 + 標籤」選取模式

## [節點 03] 2026-07-02 — 查核獨立性與下拉選單化
- 新增申請時隱藏回報區塊，需點選既有紀錄才能填寫回報
- 稽核人員與申請人相同時跳出提醒
- 分包商、人員等欄位改為下拉選單，支援「＋ 新增選項」
- 頁首新增「目前填報工程師」快速帶入申請人欄位
- 清單新增明確的編輯／刪除按鈕

## [節點 02] 2026-07-01 — 點工申請/回報架構重整
- 頁籤由 8 個精簡為 5 個，移除交接簽單、鐵板盤點模組
- 叫工申報/現場稽查更名為點工申請/回報，機具查核拆為機具申請/回報
- 申請與回報合併為單一紀錄（對齊 Excel 同一列的欄位結構）

## [節點 01] 2026-07-01 — 初版原型建置
- 依會議紀錄與既有查核表設計，建立第一版 8 頁籤稽核 APP 原型
