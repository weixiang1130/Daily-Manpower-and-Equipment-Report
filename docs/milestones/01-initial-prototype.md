# 節點 01：初版原型建置

**日期**：2026-07-01
**狀態**：已被節點 02 取代（架構已大幅調整，本文件保留作歷史紀錄）

## 背景

成本控制人員需要一套工地端可以順暢回報「點工出工」與「機具出工」的抽查稽核工具，取代目前僅靠 Excel 人工登打的方式。設計前先分析了兩份既有素材（內部文件，未收錄於本程式碼庫）：

- 稽核討論會議紀錄：歸納出的稽核設計原則
- 現行 Excel 查核表：點工出工列控 / 機具出工列控 兩分頁的實際欄位結構

## 會議紀錄歸納出的設計原則

- 現場抽查**不預告**，需以勾選為主，減少現場人員手動輸入的藉口
- 三道查核依據：**人臉辨識紀錄 ‧ 白卡進出紀錄 ‧ 工具箱會議簽名**
- 現場清點人數限時 **10 分鐘**
- 結論欄位限制在 **20 字內**，僅需簡短註記
- 簽單繳回期限放寬為抽查後 **2 天內**
- 支援工程師交接班（A 工程師到 17:00、B 工程師接續）填寫交接單
- 機具查核重點：型號、數量、是否為合約廠商，機具不需要值夜班可要求當天繳回簽單
- 鐵板盤點屬**不定期抽查**，非例行盤點，建議頻率每 2 個月一次
- 各專案可自訂固定的下拉選單清單

## 本次建置內容

建立 8 個頁籤的靜態網頁原型（純 HTML/CSS/JS + localStorage，無需伺服器）：

1. **總覽**：稽查次數、異常件數、待繳回簽單提醒、鐵板盤點提醒、最近稽查紀錄
2. **叫工申報**：工程師前一天填寫，勾選式表單
3. **現場稽查**：核心稽查表單，含 10 分鐘計時器、三道查核勾選、結論字數限制
4. **交接簽單**：A/B 工程師交接記錄
5. **機具查核**：型號/數量/合約廠商勾選、當天繳回勾選
6. **鐵板盤點**：不定期抽查框架，應有 vs 實盤差異自動計算
7. **歷程報表**：各模組表格化 + CSV 匯出，欄位對應既有 Excel 查核表
8. **設定**：每個工地可自訂固定選單（分包商/地點/內容分類）

## 資料模型（初版）

`叫工申報` 與 `現場稽查` 為**兩個獨立的陣列**（`requests[]` / `audits[]`），彼此透過下拉選單手動配對（稽查時選擇對應哪一筆叫工申報，帶出需求人數）。

## 已知限制（後續節點改善）

- 申請與回報是分開的兩張表、兩個清單，須手動配對，操作不夠直覺
- 交接簽單、鐵板盤點在使用者確認後其實用不到
- 名稱類欄位（申請人、稽核人員）用 `<input list="datalist">` 而非真正下拉選單，允許自由輸入
- 工作地點/工作內容類別為 checkbox 群組，仍需要「點選」而非下拉操作
