# 節點 02：點工申請/回報架構重整

**日期**：2026-07-01
**狀態**：已被節點 03、04 進一步調整（架構延續，欄位與互動方式已變）

## 背景（使用者原始需求）

1. 「叫工申報/現場稽查」應改名為「點工申請/出工回報」
2. 考慮是否要能針對填報的資料逐筆單獨修改/回報
3. 交接簽單可以先不用
4. 機具查核也要拆成「機具申請」跟「機具回報」
5. 鐵板盤點先不用
6. 要能點到某一筆，針對該筆進行修改，上方填寫區塊要能反映該筆內容以進行修正
7. 評估申請與回報是否要拆成兩個畫面，盡量整合到一個頁面

## 設計決策

節點 01 的 `requests[]` / `audits[]` 兩個獨立陣列，本質上對應到既有 Excel「點工出工列控」分頁的**同一列**（`需求工數` 與 `簽單實際出工數` 本來就在同一行）。因此決定**合併為單一紀錄**，用 `status` 欄位（`待回報` / `已回報`）標記回報階段是否完成，而不是維護兩個需要手動配對的陣列。

點選清單中任一筆即可載入同一張表單進行編輯——同時涵蓋申請與回報欄位，這樣比起「兩個獨立畫面 + 額外配對步驟」更貼近使用者期望的單頁整合。

## 變更內容

- 導覽列由 8 個頁籤精簡為 5 個：總覽 / **點工申請/回報** / **機具申請/回報** / 歷程報表 / 設定
- 移除交接簽單、鐵板盤點頁籤與相關程式碼
- `點工申請/回報`、`機具申請/回報` 各自變成單一表單，表單內以「申請資料」「出工回報（可稍後再填）」兩個區塊呈現，皆可編輯
- 清單新增「編輯」與「刪除」按鈕（點擊編輯會捲動並將完整資料載入上方表單）
- CSV 匯出欄位對應合併後的紀錄結構（申請與回報欄位同一列）

## 影響範圍

- `app.js`：資料模型改為 `labor[]` / `equipment[]`，STORAGE_KEY 由 `v1` 升為 `v2`
- `index.html`：移除交接簽單、鐵板盤點區塊

## 已知限制（後續節點改善）

- 申請與回報仍在**同一張表單**內（用區塊隱藏/顯示切換），理論上使用者可以一次把申請與回報一起填完，失去查核獨立性的意義
- 工作地點/工作內容類別仍是 checkbox 群組
- 申請人、稽核人員仍是 `datalist` 自由輸入
