# 節點 04：Excel 欄位對齊與大頁籤重構

**日期**：2026-07-02
**狀態**：現行架構基礎（節點 05 在此基礎上擴充多工地支援）

## 背景（使用者原始需求）

1. 「點工申請/回報」應該是一個大頁籤，底下拆分成「申請」跟「回報」兩個模組。申請維持現行做法即可；回報部分需參照現行 Excel 查核表（內部文件，未收錄於本程式碼庫）「點工出工列控」分頁表頭：`簽單繳回日、簽單實際出工數、差異、簽單責任工程師、實際工作時數(白天)、實際加班時數(晚上)、根基自辦、廠商代辦、現場查核回饋`
2. 「機具申請/回報」同樣拆成兩個模組，回報欄位參照「機具出工列控」表頭：`簽單繳回日、機具實際工作使用時數、差異、簽單責任工程師、根基自辦、廠商代辦`
3. 移除「目前填報工程師」
4. 工作地點、工作內容類別等欄位的「下拉式選單」應該要像分包商、申請人那樣的填報模式，不是節點 03 的多列選取清單

## 設計決策

### 為什麼是「大頁籤 + 兩個子模組」而不是節點 02/03 的單一表單

既有 Excel 的「簽單實際出工數」欄位本來就與「需求工數」同一列——這點在節點 02 已經確認。但使用者這次明確要求申請與回報要是**兩個真正獨立的表單**（各自送出、各自有自己的必填驗證），而不是同一張表單裡的隱藏區塊。這比節點 03 的做法更徹底地落實「查核獨立性」：回報欄位在 DOM 上完全不存在於「新增申請」流程裡，只能透過清單的「填寫回報／編輯回報」按鈕載入既有紀錄後才看得到、填得到。

### 回報欄位改為完全對齊 Excel

拿掉節點 03 自創的「稽核人員」「簽單狀態」等欄位，改用使用者提供的正式欄位名稱：

- 點工回報：`簽單繳回日`（改為可手動輸入的日期欄位，而非自動算的到期日）、`簽單實際出工數`、`差異`（自動計算）、`簽單責任工程師`（此欄位同時也是回報表單的「送出者身份」，取代原本的全域工程師選單）、`實際工作時數(白天)`、`實際加班時數(晚上)`、`根基自辦`、`廠商代辦`、`現場查核回饋`
- 機具回報：`簽單繳回日`、`機具實際工作使用時數`、`差異`、`簽單責任工程師`、`根基自辦`、`廠商代辦`

三道查核（人臉/白卡/工具箱）保留，因為這對應 Excel 本來就有的三個獨立欄位。10 分鐘計時器保留作為輔助填寫「簽單實際出工數」的小工具，不是資料欄位本身。

### 移除「目前填報工程師」

改為申請表單有自己的「申請人」下拉、回報表單有自己的「簽單責任工程師」下拉，每次都要求主動選擇——這同時也強化了查核獨立性（不會因為頁首記住了某人身份，就順手把回報也用同一人的名字送出）。

### 下拉選單改為「單行下拉 + 標籤」模式

`<select multiple>` 換成一般單行 `<select>`（跟分包商/申請人一致的外觀），選一個值就自動加到下方的標籤列表（可點 × 移除），要選第二個就再開一次下拉選單。實作上用 `tagPickerState` 追蹤各欄位目前選了哪些值，`_picker` 命名慣例區分「用來新增的下拉選單」與「儲存實際選取結果的欄位」。

## 變更內容

- `index.html`：`點工申請/回報`、`機具申請/回報` 各自變成「大頁籤 + `.subtabs` 子頁籤（申請／回報）」結構，兩個子模組是完全獨立的 `<form>`
- `app.js`：
  - 新增 `switchSubTab()` / `initSubTabs()`
  - 點工紀錄新增欄位：`signReturnDate`、`dayHours`、`nightHours`、`selfDone`、`vendorDone`；`engineer` 改為回報表單填寫
  - 機具紀錄新增欄位：`signReturnDate`、`actualHours`、`checker`、`selfDone`、`vendorDone`；移除 `actualQty`/`returnedSameDay`/`conclusion`（機具出工列控無這些欄位）
  - 新增 `tagPickerState` / `initTagPicker()` / `renderTagPicker()` 取代 `fillMultiSelect()`
  - 移除頁首 `currentEngineer` 相關邏輯
  - 清單每列的「編輯」按鈕拆成「編輯申請」（回申請模組修正申請資料）與「填寫回報／編輯回報」（跳轉回報模組）
- CSV 匯出欄位改為對齊 Excel 表頭順序

## 已知限制（後續節點改善）

- 工地仍是頁首全域選單，尚未做到「每筆申請各自選定工地」
- 總覽僅顯示單一（目前選定）工地的統計，無法一次看到所有工地狀況
