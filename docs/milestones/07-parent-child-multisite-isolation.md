# 節點 07：父子單覆核、智慧搜尋與多工地完全隔離（v6）

**日期**：2026-07-02
**狀態**：現行架構（本文件撰寫當下的最新版本）

## 背景（使用者原始需求，5 項業務邏輯）

1. **智慧搜尋與動態新增**：分包商、申請人、工作地點改為 Autocomplete/Combobox，輸入即模糊搜尋；搜不到時選單底部「＋ 新增選項：『輸入文字』」，所有承辦人員皆可點擊新增並寫入**該工地**的基礎資料庫
2. **點工申請(父)/回報覆核(子)階層**：回報直接承繼申請單資料；回報畫面列出申請單的預計進場人員名單，逐人**勾選**確認到場，勾選後於後方填該員實際出工狀況與加班時數；若當日完全無人出工，勾選確認後以「0 工」寫入
3. **欄位精簡**：全面移除「實際工作時數」；保留「實際加班時數」；「簽單實際出工數」支援小數（0.5、1.5 工）
4. **防呆警告**：出工數與加班時數配置異常（如 0.5 工配 8 小時加班）時，送出前彈出警告，但**不可鎖死阻擋送出**
5. **多工地完全隔離**：各工地資料庫完全隔離；明確的工地切換機制（Context Switch），切至 A 工地時所有下拉只帶 A 工地資料；每個工地預設帶入 50 家實際分包商名單

## 設計決策

### 儲存層：每工地一把獨立 key

從 v5 的單一 `STATE` 改為：

- `dm_master_v6`：工地清單 + 目前工地（Context）
- `dm_site_v6::<工地名>`：該工地的基礎資料（config）與紀錄（labor/equipment），**彼此完全獨立**

這個結構直接對應「後端資料庫完全隔離」的要求——未來接後端時，每把 key 可對應一個獨立的 collection/schema，遷移路徑清晰。工地切換（頁首下拉或總覽點選工地列）即切換整個資料環境，所有表單重置、所有下拉重新指向新工地的資料池。

### Combobox 取代原生 select

自製輕量 combobox（無外部依賴）：輸入即過濾（`includes` 匹配、`startsWith` 優先排序）、鍵盤 Enter 選第一項、Escape 關閉、blur 延遲關閉避免吃掉點擊。找不到完全相符項目時，底部固定顯示「＋ 新增選項」，點擊即寫入目前工地 config 並自動選取。適用欄位：分包商、機具廠商、申請人、簽單責任工程師、工作地點（多值+標籤）、點工人員（多值+標籤）。工作內容類別與機具類型維持原 select 型 tag picker（不在需求範圍）。

### 父子單資料結構

申請單新增 `workers[]`（預計進場人員名單，來源為各工地獨立的 `workers` 人員池，可即時新增）；需求工數隨名單長度自動同步（仍可手動修改）。回報不再是獨立表單資料，而是寫入父單的 `report` 子物件：

```
labor: {
  id, date, vendor, applicant, required, workers[], locations[], categories[], categoryNote,
  status: "待回報" | "已回報",
  report: null | {
    reportedAt, engineer, checkFace/Card/Toolbox,
    attendance: [{ name, present, work, ot }],   // 逐人勾選＋小數出工數＋加班時數
    actual, totalOT, diff, zeroWork,
    signReturnDate, selfDone, vendorDone, conclusion
  }
}
```

回報畫面逐人一列：勾選到場 → 顯示該員的出工數（預設 1，step 0.5）與加班時數輸入；總計欄位自動加總（仍可手動覆寫，對應「申請單沒填名單」的情境）。

### 0 工路徑

送出時若實際出工數為 0 且未勾「0 工確認」→ 擋下並提示（這是唯一的硬性擋，因為有明確的替代路徑）；勾選「0 工確認」會自動取消所有到場勾選並鎖定名單區，送出後寫入 `zeroWork: true`，清單與報表以紅色「0工」標籤呈現。

### 防呆警告規則（可送出）

逐人檢查：到場但出工 0／出工 ≤0.5 但加班 >4h／加班 >8h／單人出工 >2 工；總量檢查：加班總時數 > 出工數 × 8。命中任一規則時以 `confirm()` 彈窗列出全部異常明細，使用者確認後仍可送出（不鎖死）。

## 變更內容

- `app.js` 全面改寫（v6）：儲存層、combobox、父子單、attendance、警告引擎、跨工地總覽
- `index.html`：申請/回報表單改用 combobox 容器；回報表單新增出工人員覆核區與 0 工確認；移除「實際工作時數(白天)」欄位；設定頁拆分「全域工地清單」與「目前工地基礎資料」（含點工人員池）
- `style.css`：新增 `.combobox/.cb-*`、`.attendance-box/.att-*`、`.zero-chip` 樣式
- `config.local.js`（僅地端）：vendors 更新為 50 家實際分包商，作為每個工地建立時的預設名單
- CSV 點工報表：移除白天時數欄，新增「預計進場人員」「0工確認」「出工人員明細」（如：張三(1工/加班2h)、李四(0.5工)）

## 驗證紀錄

- 模糊搜尋：輸入「鋐」正確過濾出 4 家；無符合時只顯示「＋ 新增選項」
- 新增選項寫入目前工地資料庫，切換工地後不外漏（A 工地新增的廠商/人員 B 工地看不到）
- 父子流程：申請（3 人名單）→ 回報勾 2 人、其中 1 人 0.5 工＋8h 加班 → 警告彈窗列出異常 → 確認後成功寫入
- 0 工：未勾確認送出被擋；勾選後以 0 工寫入並正確標示
- 小數：1.5 工加總、-1.5 差異顯示正常
- 每工地 50 家預設分包商正確種入；16 工地各自獨立

## 已知限制

- 「承辦人員權限」目前無登入機制，等同全開放（符合現階段需求）；未來接後端後可加角色控管
- 隔離目前是同一瀏覽器內的邏輯隔離；跨裝置/多人協作仍需後端資料庫
- v5 及更早的測試資料不做遷移（原型階段，資料結構差異大）
