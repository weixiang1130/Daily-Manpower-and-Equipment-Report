# 節點 08：機具父子覆核、移除計時器與管理員模式（v7）

**日期**：2026-07-02
**狀態**：現行架構（本文件撰寫當下的最新版本）

## 背景（使用者原始需求）

1. 現場清點計時（10 分鐘倒數）不需要了，刪除
2. 機具查核回報也要和點工一樣有父層/子層系統
3. 詢問部署到 Netlify 需要提供什麼（另行回覆，不在本節點程式變更內）
4. 部署後不同工地需要權限控管：透過簡單設計讓「清除功能」只在管理員身分時顯示，一般人仍可新增資料；另外清空工地按鈕太大容易誤觸，要縮小

## 變更內容

### 移除現場清點計時器
- 刪除回報表單的 10 分鐘倒數計時 UI 與全部相關程式碼（`auditTimer`／`startAuditTimer`／`resetAuditTimer`／`renderTimerDisplay`）與樣式

### 機具回報改為父子覆核（與點工一致）
- 申請單（父層）的機具類型清單，在回報（子層）畫面逐台列出：勾選「到場」後填寫該台的「實際工作使用時數」（step 0.5、預設 8）
- 總計時數自動加總（仍可手動覆寫）；差異照舊 = 實際時數 − 需求數量
- 「0 使用確認」：機具完全未到場／未使用時，勾選後以 0 時數寫入；未勾選而時數為 0 直接送出會被擋下（與點工 0 工邏輯一致）
- 防呆警告（不阻擋送出）：勾選到場但時數 0／單台單日逾 12 小時
- 資料結構：`rec.report.usage = [{type, present, hours}]`、`zeroUse` 旗標；清單顯示紅色「0時數」標籤；CSV 新增「0使用確認」「機具使用明細」欄（如：吊車(6h)、挖土機(4h)）

### 管理員模式（前端層級防誤觸）
- 設定頁新增「身分模式」列：預設為一般使用者，點「管理員登入」輸入密碼切換（存於 sessionStorage，關閉分頁即登出）
- 密碼來源：`config.local.js` 的 `adminPin`（程式碼庫預設範例值 `0000`，地端請自行設定）
- 一般使用者：工地清單與各基礎資料 textarea 唯讀、儲存/還原按鈕隱藏、資料管理（清空）區塊整個隱藏；**表單搜尋欄位的「＋ 新增選項」不受影響**，人人可用
- 管理員：全部功能開放；各管理區塊標題附「管理員」徽章
- 除 UI 隱藏外，事件處理器內也有 `isAdmin()` 守衛（雙重防護）
- 清空按鈕改為小尺寸（`btn-danger-sm`），降低誤觸

## 設計決策與誠實限制

管理員模式是**前端層級的防誤觸設計，不是資安防線**——技術上任何人開瀏覽器開發者工具都能繞過（資料本來就在使用者的 localStorage 裡）。它解決的問題是：現場一般人員不會誤觸清空、不會隨意批次改名單。若需要真正的帳號權限（不同工地不同帳號、伺服器端驗證），需接後端服務（如 Supabase Auth），屆時每個工地的 store 對應一個受權限保護的資料表。

## 驗證紀錄

- 計時器 UI 與程式碼完全移除，console 無錯誤
- 機具父子流程：申請 2 類型（吊車＋挖土機）→ 回報勾吊車 6h → usage 明細正確、總計 6、差異 +5
- 0 使用：未勾確認送出被擋、勾選後以 0 時數寫入並顯示紅色標籤
- 管理員：預設唯讀＋清空區隱藏；錯誤密碼被拒；正確密碼後全部開放；登出恢復唯讀
