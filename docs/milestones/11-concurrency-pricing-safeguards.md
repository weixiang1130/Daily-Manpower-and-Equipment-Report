# 節點 11：共編保險絲、計價期間篩選與營運保障（v9）

**日期**：2026-07-03
**狀態**：現行架構（本文件撰寫當下的最新版本）

## 背景（使用者原始需求）

共編上線後進行通盤盤點，使用者裁示：
1. 匯出維持「目前工地」範圍即可（成本部承辦各管各的專案，自行切換工地抓取）
2. 加入日期區間篩選，方便按月計價
3. 第二批（共編正確性）與第三批（營運保障）執行；第一批其餘項目（狀態快篩、待回報面板、逐人計價表）暫緩再議

## 修正：回報日期時區 Bug

`toISOString()` 取的是 UTC 日期——台灣時間早上 8 點前填寫的回報會被記成**前一天**，月底/月初的單會被歸錯計價月份。改以本地時區組日期字串（`localDate()`），回報日、CSV 檔名、備份檔名一體適用。此為資料正確性修正，不在功能取捨範圍。

## 共編保險絲（三道）

| 機制 | 做法 | 擋掉的事故 |
|---|---|---|
| 編輯前抓最新 | 點「編輯申請／填寫回報」時先 `GET ?site=` 重抓該工地資料再帶入表單；紀錄已被刪除則提示 | 用早上開著沒動的舊畫面去改單，把別人上午的修改蓋掉 |
| 版本衝突偵測 | 每筆紀錄帶版本號 `v`；寫入時附 `baseV`，伺服器版本不符回 409，前端提示「已被他人修改，請重新確認」並重載 | 兩人真的同時編輯同一筆，後送出者無聲覆蓋 |
| 選項伺服器端合併 | 「＋新增選項」改走 `op:addOption`，由伺服器讀取-合併-寫回單一選項 | 兩人同時各新增一家分包商，後寫的整包名單裡沒有對方剛加的 |

409 也涵蓋「編輯中被刪除」（`existing==null && baseV>0`）。設定頁的批次儲存仍為整包覆蓋（管理員限定、頻率低，可接受）。

## 計價期間篩選

歷程報表新增期間列：起迄日期＋「本月／上月／全部」快捷鈕；表格與 CSV 同步套用，顯示「共 N 筆（區間）」，CSV 檔名附上區間。範圍仍為目前工地（依使用者裁示）。

## 營運保障（三項）

1. **已回報單刪除保護**：已回報的單據是計價依據，非管理員不可刪除
2. **計價鎖定日期（鎖單）**：設定頁（管理員）可設每工地的 `lockDate`；該日期「含」以前的單據，非管理員不可新增／編輯／回報／刪除——月結匯出後鎖住該期間，計價依據不再變動。屬前端層級管控（與管理員模式同一信任等級）
3. **完整備份下載**：設定頁（管理員）一鍵下載全部工地資料的 JSON 快照，防清空誤操作／供異地留存

## 驗證紀錄

- 本機（狀態注入）：期間篩選與本月/上月/全部快捷正確過濾筆數；鎖定日期在設定頁正確呈現且非管理員 disabled；已回報單刪除、鎖定期間刪除、鎖定日期申請送出（在工地確認視窗之前）、非管理員備份，四種守衛皆正確擋下並提示
- production（部署後 curl，拋棄式測試工地、驗畢清除）：連續寫入版本正確遞增（baseV 0→v1、1→v2、2→v3）；過期 baseV 回 409；刪除後以 baseV>0 寫入回 409（編輯中被刪除情境）；addOption 連續新增兩選項皆保留（合併正確）

## 已知限制

- 鎖單與刪除保護為前端管控，懂技術者可繞過（與管理員模式相同的既有信任模型）；伺服器端強制需搭配真正的登入機制
- 版本衝突提示後，使用者在表單中已輸入的內容不保留（需重填）；因「編輯前抓最新」已大幅降低衝突發生率，屬可接受的簡化
