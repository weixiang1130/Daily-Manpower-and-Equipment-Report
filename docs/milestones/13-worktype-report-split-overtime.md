# 節點 13：回報改逐工種、分段加班與彙總期間欄（v11）

**日期**：2026-07-07
**狀態**：現行架構（本文件撰寫當下的最新版本）

## 背景（使用者現場回饋，四點照單優化）

系統實際上線後，現場使用回饋：
1. 「出工人員名單」用不到——申請的工程師不會知道點工姓名，現場工程師也不認得，逐人覆核形同虛設
2. 工地點工分「粗工／技術工／打石工」等工種，計價單價不同；申請時不分，但回報覆核必須分工種記錄
3. 匯出的計價彙總 CSV 沒有日期資訊，承辦無法辨識彙總涵蓋期間
4. 點工加班費率分段：前 2 小時一個價、第 3 小時起另一個價，回報時必須分開填

## 變更內容

### 1. 申請端：移除人員名單
- 「預計進場人員名單」欄位整個移除；「需求工數」改純手填（原本隨名單自動計算）
- 舊單資料相容：編輯舊申請單時原 `workers` 陣列原樣保留，不會被清掉

### 2. 回報端：逐人覆核 → 逐工種覆核
- 「出工人員覆核」改為「出工工種覆核」：搜尋下拉（新名單池 `laborTypes`，預設 粗工／技術工／打石工／其他，可「＋新增選項」）選工種加入一列，逐列填 出工數＋分段加班；列可 × 移除，重複工種擋下
- 有工種明細時「簽單實際出工數」與兩個加班總計欄自動加總（唯讀）；無明細時開放手填（相容舊單）
- v10 的「補入出工人員」機制隨逐人覆核一併移除

### 3. 加班分段（前 2 小時／第 3 小時起）
- 每工種列與總計欄皆拆「加班·前2小時」「加班·第3小時起」；`totalOT` 保留為兩段合計（清單欄位與相容用）
- 防呆新規則：前 2 小時加班 > 出工數×2 小時 → 警告；填了第 3 小時起但前 2 小時為 0 → 警告；合計 > 出工數×8 照舊
- 資料結構：`report.workTypes[{type,work,ot2,otOver}]`、`ot2Total`、`otOverTotal`；舊單 `attendance` 原樣保留不遷移

### 4. 報表與計價彙總
- 明細 CSV：移除「預計進場人員」欄；「實際加班時數(晚上)」拆為「加班時數(前2小時)」「加班時數(第3小時起)」「加班總時數」三欄；明細欄改「出工明細(工種)」——新單顯示 `粗工(3工/前2h:6h/逾2h:3h)`，舊單 fallback 顯示逐人明細
- 計價彙總（畫面＋CSV）：**新增「期間」欄**（起~迄或「全部期間」），加班拆兩欄；舊單的 totalOT 計入前 2 小時段
- 後端 `op:addOption` 白名單加入 `laborTypes`；設定頁批次編輯區新增「出工工種清單」，移除「點工人員名單」輸入區（`workers` 資料保留於資料庫不動）

## 設計決策 / 取捨

- **工種為每工地獨立名單池**：與其他選項一致，各工地可自行增補；既有 16 個工地的 config 缺此欄位，於前端載入時自動補預設值（首次在該工地「＋新增選項」或管理員儲存設定時才寫回雲端）
- **舊單不遷移**：舊逐人資料保留原欄位，讀取端 fallback；舊單編輯時總數欄開放手填、舊 totalOT 帶入「前 2 小時」欄（實務上舊加班多屬前段），彙總同樣歸前段並於文件註明
- **工種列不設到場勾選**：列出即到場（不到場就不要加），比照人數勾選邏輯簡化

## 影響範圍

- `index.html`：點工申請（移除名單欄）、點工回報（工種覆核區、分段加班欄）、設定頁（工種清單 textarea）
- `app.js`：GENERIC_CONFIG/defaultSiteConfig/sortRecords（laborTypes 池與補值）、申請表單、回報表單全段（typeState/renderTypeRows/addTypeRow/syncTotals/警告）、laborDetail、REPORT_DEFS.labor、計價彙總（期間欄＋分段）、SITE_CFG_MAP
- `netlify/functions/api.mjs`：addOption POOLS 白名單 +laborTypes
- 資料結構：`report` 新增 `workTypes/ot2Total/otOverTotal`（向下相容）；`config` 新增 `laborTypes`

## 驗證紀錄

本機（狀態注入＋fetch 快照 stub，未觸碰共用資料庫）：
- 新單逐工種：加入粗工/技術工、重複擋下、逐列填數自動加總唯讀（5工/前2h 6/逾2h 3）、送出寫入 workTypes 與 ot2Total/otOverTotal/totalOT=9、差異 0
- 舊單（僅 attendance/totalOT）：載入為手填模式、totalOT 帶入前 2 小時欄；明細 CSV fallback 逐人明細、加班分段欄空白、總計欄有值
- 防呆：前 2 小時 > 出工數×2 觸發列級與總計級警告
- 明細 CSV 新表頭與新舊單各欄正確；彙總含期間欄與分段加班、舊單歸前段
- 申請表單無名單欄位送出正常（workers 存空陣列）；設定頁出現工種清單、無點工人員區塊；缺 laborTypes 的 config 載入自動補預設
