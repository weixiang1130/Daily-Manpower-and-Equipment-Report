# 節點 17：移除根基自辦欄位、查核回饋取消字數限制（v12）

**日期**：2026-07-07
**狀態**：已上線（PR #18 合併於 2026-07-09，使用者核准部署。部署後即以 HTTP 檢查正式站四項標記通過：兩表單自辦欄位移除、回饋無 maxlength、標籤無註記；本文件下方「驗證紀錄」為部署前的本機測試）

## 背景（使用者需求，兩點）

1. 回報時已填「廠商代辦」，剩下的必然是自辦——兩者都填浪費時間，移除「根基自辦」只留「廠商代辦」
2. 「現場查核回饋」不需限制字數

## 變更內容

### 1. 移除「根基自辦」輸入欄位（點工＋機具回報表單一致）
- 兩表單各移除 根基自辦（工數/時數/備註）三欄；廠商代辦（工數）placeholder 加註「未填代辦＝全數自辦」
- **舊資料保留**：編輯舊單重新儲存時，既有 selfDone* 值原樣承繼寫回（不會因表單無欄位而被清空）；舊版單一文字欄 selfDone 亦歸入 selfDoneNote 承繼
- **報表不動**：明細 CSV 與計價彙總的「根基自辦」欄位保留（舊資料仍完整呈現，新單為空白/0）

### 2. 現場查核回饋取消字數限制
- 移除 textarea 的 maxlength（原 30）與字數計數器，label 改「選填，不限字數」，rows 2→3
- 順手清除失去呼叫者的 bindCharCounter 函式（死碼）

### 3. 文件同步（合約變更紀律）
- `docs/API-CONTRACT.md`：selfDone* 註記「v12 起表單移除、寫入原樣承繼、僅舊資料非空」；conclusion 改不限字數並提醒後端欄位長度（TEXT 級）

## 設計決策 / 取捨

- **機具表單一併移除**：使用者敘述的邏輯（有代辦、剩下即自辦）對兩表單同樣成立，且維持 UI 一致；若僅點工需要可單獨還原機具（一個 commit 內可辨識）
- **承繼而非刪除舊值**：selfDone* 欄位轉為「唯讀歷史資料」而非資料清洗——計價歷史不可變動
- **CSV 欄位保留**：成本部門對過往月份對帳仍需舊自辦數字

## 影響範圍

- `index.html`：兩回報表單（各 -3 欄）、查核回饋欄
- `app.js`：兩表單 submit（selfDone* 承繼）、load（移除欄位帶入）、reset/計數器清理
- `docs/API-CONTRACT.md` 同步
- 後端（api.mjs / server.mjs）**零修改**——合約形狀不變

## 驗證紀錄

本機（狀態注入＋fetch stub，未碰共用資料庫）：
- 新單回報送出：report 內 selfDone* 為 null/空字串、vendorDone* 正常寫入
- 帶舊自辦資料（selfDoneWork=2、selfDoneHours=4、舊字串 selfDone）的已回報單 → 編輯重存 → selfDone* 三值原樣保留、selfDone 字串歸入 Note
- 查核回饋輸入 120 字長文 → 完整寫入不截斷；UI 無計數器、無 maxlength
- 兩表單畫面確認無自辦欄位、代辦 placeholder 更新；console 無錯誤
