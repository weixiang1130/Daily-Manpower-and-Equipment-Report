## 如何新增下一個節點

每次改版完成後，請新增一份 `0N-描述.md`，內容至少包含：
1. **背景**：這次改版要解決什麼問題（可引用使用者原始需求）
2. **變更內容**：條列實際改了什麼
3. **設計決策 / 取捨**：為什麼這樣做，有沒有考慮過其他做法
4. **影響範圍**：牽動哪些檔案、資料結構是否變動

並在本頁表格新增一列連結，同時更新根目錄 `CHANGELOG.md`。
